## Basic Usage

### 1. Assistant returns structured JSON

This JSON can be generated by **any LLM**.

```json
{
  "role": "assistant",
  "content": "I need one more detail before continuing.",
  "ui": {
    "id": "priority_level",
    "type": "single_choice",
    "label": "How urgent is this?",
    "options": [
      { "id": "low", "label": "Low priority" },
      { "id": "medium", "label": "Medium priority" },
      { "id": "high", "label": "High priority" }
    ],
    "required": true
  }
}
```

---

### 2. Render the assistant message

```tsx
import { AssistantMessage } from "@chatforms/react"

<AssistantMessage message={assistantMessage} />
```

ChatForms renders the text **and** the interactive UI inline.

---

### 3. Serialize user selections on submit

```ts
import { serializeSelections } from "@chatforms/react"

const payload = serializeSelections({
  userInput: "This blocks a customer launch",
  selections
})
```

Result:

```json
{
  "role": "user",
  "content": "This blocks a customer launch",
  "structured_selections": {
    "priority_level": "high"
  }
}
```

This payload is what you send to your LLM.

## Supported UI Blocks (v0)

- `single_choice` (buttons)
- `multi_choice`
- `dropdown`
- `input` (short text)
- `confirm` (yes / no)

The system is schema-driven and extensible.

---

## Schema Overview

Each assistant message may include an optional `ui` block.

```ts
interface ChatFormBlock {
  id: string
  type: "single_choice" | "multi_choice" | "dropdown" | "input" | "confirm"
  label?: string
  options?: {
    id: string
    label: string
  }[]
  required?: boolean
}
```

See `/packages/core/schema.ts` for the canonical definition.

## Design Principles

- **Chat first** â€“ UI enhances conversation, never replaces it
